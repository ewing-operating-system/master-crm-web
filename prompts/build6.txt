Steps:

   - Look for lib/*.py files in ~/Projects/master-crm-web/
   - Understand how they read from Supabase and write HTML to public/

2. Create scripts/regenerate.py:
   - Connects to Supabase (dwrnfpjcvydhmhnvyzov, credentials from ~/.openclaw/.env)
   - Queries for targets with status = 'RESEARCHED' and no generated page (or updated research newer than page)

   - Updates target status to 'PUBLISHED' in Supabase
   - Logs what was generated

3. Create scripts/regenerate_all.py:
   - Regenerates ALL pages (not just new ones) — useful for template changes
   - Same pattern but iterates all targets regardless of status

4. Verify the script works by checking:
   - Can it connect to Supabase?
   - Can it read target data?
   - Can it generate at least one page?
   - Does git push work? (check git remote is configured)
